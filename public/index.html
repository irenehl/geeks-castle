04

Parte técnica

src/ · Clean Architecture

Placa de especificación del user-service: cómo está armado el backend detrás de esta consola.

Stack
NestJS · TypeScript · Firebase / Firestore (Admin SDK) · Clean Architecture · bcrypt · @nestjs/event-emitter
Capas
  • domain/ - entidad User, contrato del repositorio, evento de dominio
  • application/ - casos de uso, generador de contraseña, handler de user.created
  • infrastructure/ - Firebase Admin + repositorio Firestore
  • presentation/ - controllers, DTOs, módulos Nest
Flujo
Crear usuario > evento user.created > si falta contraseña, generar una segura > hash bcrypt > actualizar documento en Firestore
Endpoints
  • POST /users - añadir usuario
  • GET /users/:id - consultar por UUID
  • GET /users - listar todos
Datos
Colección Firestore users. Emulador local si FIREBASE_USE_EMULATOR=true (host en FIRESTORE_EMULATOR_HOST); producción con credenciales de servicio cuando el emulador está desactivado (ver .env).
Validaciones
  • username - 3-32 chars; [a-zA-Z0-9_]
  • email - formato válido; máx. 254
  • password - opcional; si viene: 8-128 chars con mayúscula, minúscula, dígito y símbolo
  • :id - UUID v4 ( GET /users/:id)
Configuración
# Emulador Firestore
npm run emulator

# API + consola (http://localhost:3000)
npm run start:dev

Emulador UI: http://localhost:4000
